01 — Foundation

Structure vs Function — The Two Key Terms

Before anything else, your exam asks this exact question: "Distinguish between Structure and Function." Let's understand it deeply, not just memorize it.

🏗️ STRUCTURE
How components are related to each other. The physical arrangement — what connects to what.
⚙️ FUNCTION
What each component does. The operation or behaviour of each part individually.
🏠 Real World Analogy

Think of a house. The structure is how the rooms, walls, doors, and pipes are connected. The function is what each room does — the kitchen cooks food, the bedroom is for sleeping. A computer is the same: structure = wiring & layout, function = what each part does.

03 — The Four Functions

What Can a Computer DO? (4 Basic Functions)

Every computer, from the oldest to the newest, can only do four fundamental things:

1. DATA PROCESSING
Transform data — add numbers, compare values, run logic. The CPU does this.
2. DATA STORAGE
Remember data — short-term (RAM/cache) or long-term (hard drive).
3. DATA MOVEMENT
Transfer data between components — from memory to CPU, from I/O to memory, etc.
4. CONTROL
The Control Unit manages and coordinates all the above — it's the "manager" of the CPU.
🍳 Kitchen Analogy

A kitchen does the same four things: it processes raw ingredients into food, stores food in the fridge, moves ingredients from fridge to stove, and the chef controls everything. Your CPU is just a very fast, very precise kitchen.

04 — Think Deeper

Why Does This Matter For The Exam?

Your 2018 exam asked: "Distinguish between Structure and Function, then draw and describe the general structure and functions of an I/O module."

🧠 Brain Challenge — Think Before You Read

Before I show you the answer: Based on what you've learned, what do you think the function of an I/O module is? Write your thoughts below.

The I/O module has these functions:

1. Control & Timing — coordinates data flow between external devices and the internal bus.
2. CPU Communication — receives commands from the CPU.
3. Device Communication — sends/receives data to/from external devices (keyboard, disk, etc).
4. Data Buffering — temporarily stores data to handle speed mismatches between CPU and slow I/O devices.
5. Error Detection — detects transmission errors.
